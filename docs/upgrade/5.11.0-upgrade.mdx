### 控制台升级

<Tabs groupId="upgrade">
  <TabItem value="helm" label="helm" default>

1. 执行 sql 升级脚本

- 进入 Rainbond-控制台组件

```bash
kubectl exec -it $(kubectl get po -n rbd-system | grep rbd-app-ui | grep Running | sed -n '1p' | awk '{print $1}') -n rbd-system bash
```

- 进入 Rainbond-控制台组件后，执行以下命令

```bash
curl https://rainbond-script.oss-cn-hangzhou.aliyuncs.com/upgrade-5.11.0.sh | bash
```

  </TabItem>
  <TabItem value="Basic Edition" label="Basic Edition">

1. 执行 sql 升级脚本

- 如控制台为 allinone 启动，执行以下操作

```bash
docker exec -it rainbond-allinone bash
curl https://rainbond-script.oss-cn-hangzhou.aliyuncs.com/upgrade-5.11.0.sh | bash
```

- 如控制台通过应用市场安装，进入 Rainbond-控制台组件后，执行以下命令

```bash
curl https://rainbond-script.oss-cn-hangzhou.aliyuncs.com/upgrade-5.11.0.sh | bash
```

2. 升级控制台

- 如控制台为 allinone 启动，执行以下操作

```bash
docker stop rainbond-allinone && docker rm rainbond-allinone
#该命令参数需要和之前启动的rainbond-allinone容器一致
docker run -d -p 7070:7070 -v ~/.ssh:/root/.ssh -v ~/rainbonddata:/app/data \
--name=rainbond-allinone --restart=always \
registry.cn-hangzhou.aliyuncs.com/goodrain/rainbond:v5.11.0-release-allinone
```

如控制台为通过应用市场安装，进入控制台应用内，选择升级到 5.11.0 版本
</TabItem>
</Tabs>

###  数据中心升级

<Tabs groupId="upgrade">
  <TabItem value="helm" label="helm" default>

- 数据中心端升级

- 更新 grctl 命令

```bash
docker run -it --rm -v /:/rootfs  registry.cn-hangzhou.aliyuncs.com/goodrain/rbd-grctl:v5.11.0-release copy
mv /usr/local/bin/rainbond-grctl /usr/local/bin/grctl && grctl install
```

**注意：**如果遇到无法安装 grctl 可以[参考grctl安装文档](/docs/ops-guide/tools/grctl/)

- 执行升级命令

```bash
#替换基础 region 镜像版本
grctl cluster upgrade --new-version=v5.11.0-release
#手动替换 operator 镜像版本为 v5.11.0-release
kubectl edit deploy rainbond-operator -n rbd-system
```

  </TabItem>
