# install-hub 设计记录

## 目标

把 `install-hub` 从“安装入口集合页”调整为“安装决策引导页”，帮助用户更快判断自己当前场景，并进入正确的安装路径。

## 设计结论

- 采用更克制的首屏，不使用过大的标题。
- 先给出 3 个高频判断条件：
  - 当前是试装 / PoC，还是正式部署
  - 是否已有可用 Kubernetes 集群
  - 是否存在离线、内网或信创限制
- 页面主体按决策顺序重组，而不是把所有入口等权平铺。

## 信息架构

### 1. 首屏判断区

- 简洁标题与说明
- 3 个快速判断标签
- 右侧判断提示，帮助用户快速建立分流心智

### 2. 首次安装主路径

首次安装只保留 4 条核心主路径：

- 快速安装
- 基于主机安装
- 基于 Kubernetes 安装
- 离线安装

每张卡片展示：

- 适合场景
- 关键前置条件
- 典型投入或工具
- 直接跳转 CTA

### 3. 非首次安装 / 特殊场景入口

- 多集群安装：强调这是在已有控制台后的扩展动作
- 信创专题：强调这是专题入口，不与首次安装主路径混淆

### 4. 安装后动作

- 部署第一个应用
- 排障与支持

## 展现原则

- 用分组、间距和对比建立层级，不依赖超大标题。
- 文案更短，优先表达“你是谁、你该去哪”。
- 页面既保留文档站的阅读感，也具备引导页的分流能力。
