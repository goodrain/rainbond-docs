## 操作步骤
<TrackedDetails summary="了解安装原理" section="quick_install_principle">

**快速安装模式的工作原理**

在快速安装模式下，Rainbond 采用了一种简化的架构设计：

1. **单容器部署**：整个 Rainbond 平台被封装在一个 Docker 容器中，包含了所有必要的组件
2. **内置 Kubernetes**：容器内部运行了一个轻量级的 Kubernetes 集群，你无需手动安装和配置

**与传统部署方式的区别**

| 特性 | 快速安装模式 | 生产环境部署 |
|------|-------------|-------------|
| **复杂度** | 一键安装，无需了解 K8s | 需要了解 K8s 或使用安装向导 |
| **资源占用** | 轻量级，适合体验和测试 | 完整功能，适合生产环境 |
| **扩展性** | 单机部署，单节点 | 集群部署，多节点 |
| **适用场景** | 学习、测试、体验 | 生产环境、团队协作 |

**快速安装相关名词**

- **EIP（External IP）**：外部访问 IP（内网/公网IP），用于从外部访问 Rainbond 控制台和部署的应用

在 Linux 上执行以下命令，该脚本会通过 Docker 快速启动 Rainbond。也支持在国产化信创的环境中运行。

<CopyCommandBlock
  code={`curl -o install.sh https://get.rainbond.com && bash ./install.sh`}
  osTab="linux"
  commandType="quick_install"
  hint="复制后直接在终端执行，适合 Linux 单机安装。"
/>

执行完上述脚本后，耐心等待 2-3 分钟，直到出现如下提示，即表示安装成功：

```bash
🎉 Rainbond Installation Successful!
......
OS: Linux
Access Rainbond:
    🌐 Console: http://192.168.1.1:7070
```

  </TabItem>

:::warning 注意
由于 Rainbond 需要运行内置的 Kubernetes，而 Docker Desktop 在 MacOS 上存在虚拟化嵌套限制，因此需要使用 OrbStack 替代。OrbStack 是更底层的容器运行时，与 MacOS 兼容性更好，且完全兼容 Docker 命令，使用方式不变。
- 请在 MacOS 上安装并启动 [OrbStack](https://orbstack.dev)
:::

在 MacOS 终端上执行以下命令，该脚本会通过 OrbStack Docker 快速启动 Rainbond。

<CopyCommandBlock
  code={`curl -o install.sh https://get.rainbond.com && bash ./install.sh`}
  osTab="mac"
  commandType="quick_install"
  hint="复制后直接在 OrbStack 环境里执行。"
/>

执行完上述脚本后，耐心等待 2-3 分钟，直到出现如下提示，即表示安装成功：

```bash
🎉 Rainbond Installation Successful!
......
OS: Darwin
Access Rainbond:
    🌐 Console: http://192.168.1.1:7070
```

  </TabItem>

## 成功标准

- 脚本执行完成后输出 `Rainbond Installation Successful`。
- 浏览器可以打开 `http://你的IP:7070`。
- 你已经知道下一步要去 [部署第一个应用](./getting-started.md)，而不是停在“命令执行成功”。

本文档介绍使用脚本一键安装 Rainbond，脚本检测环境完成后会在使用 Docker 启动 Rainbond 容器。支持 `Mac(Intel/Apple Silicon)`、`Linux(x86_64、Arm64)`。
